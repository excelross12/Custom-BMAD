Step 2 DO THIS NEXT
Project Analysis & Brainstorming
What to do: Think about what you want to build. The BMAD Analyst will help you explore your idea and write it down.
Say: "Act as BMAD Analyst"
Files You'll Create:
docs/brainstorming-session.md (your ideas)
docs/project-brief.md (summary of what to build)

Need Help?

  • Open your AI assistant (Cursor, Windsurf, etc.)
  • Type: "Act as BMAD Analyst"
  • The AI will ask you questions about your project
  • Answer the questions - it helps you think clearly!
  • Save the conversation to the files shown above
Step 3 OPTIONAL
Market Research
Look at similar projects and see what makes them good or bad. This is optional but helpful!
Say: "Act as BMAD Analyst. Create research prompt"
File You'll Create:
docs/market-research.md
Step 4 COMING NEXT
Create Product Requirements (PRD)
What to do: Turn your idea into a detailed plan with all the features you want.
Say: "Act as BMAD Product Manager"
File You'll Create:
docs/prd.md (detailed feature list)
Step 5 COMING NEXT
Design User Interface (If You Have a UI)
What to do: Plan how your app will look and how users will use it.
Say: "Act as BMAD UX Expert"
File You'll Create:
docs/front-end-spec.md (UI design plan)
Step 6 COMING NEXT
Design System Architecture
What to do: Decide what technologies to use and how everything fits together.
Say: "Act as BMAD Architect"
File You'll Create:
docs/architecture.md (technical blueprint)
Step 7 COMING NEXT
Break Into Smaller Pieces (Epics)
What to do: Break your big plan into smaller, manageable chunks called "epics".
Say: "Act as BMAD Product Owner. Shard the PRD"
Files You'll Create:
docs/epics/epic-001-[name].md
docs/epics/epic-002-[name].md
...and more epics
Step 8 COMING NEXT
Create Detailed Tasks (Stories)
What to do: Break each epic into super-detailed tasks that tell you exactly what to code.
Say: "Act as BMAD Scrum Master. Create story from epic-001"
Files You'll Create:
docs/stories/story-001-[name].md
docs/stories/story-002-[name].md
...one story at a time
Step 9 COMING NEXT
Write the Code (Implementation)
What to do: Now you code! The Developer agent will help you write code, tests, and documentation - one story at a time.
Say: "Act as BMAD Developer. Implement story-001"
Files You'll Create:
src/ (your source code)
tests/ (your tests)
+ Dev Notes in each story file
Step 10 COMING NEXT
Quality Check (Testing)
What to do: Make sure everything works correctly. The QA agent will help you test thoroughly.
Say: "Act as BMAD QA. Review story-001"
What Happens:
All tests run and pass
Code is checked for errors
QA report added to story

Quick Reference: File Locations

Where Everything Goes

  • docs/ - All your planning documents go here
  • docs/epics/ - Big feature chunks
  • docs/stories/ - Detailed tasks
  • src/ - Your actual code
  • tests/ - Your test code
  • reference/ - BMAD helper files (don't change these)
